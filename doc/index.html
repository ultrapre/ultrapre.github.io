什么是虚拟应用程序？

虚拟应用程序是预配置了特定应用程序立即执行所需的所有文件，注册表数据，设置，组件，运行时以及其他依赖项的虚拟机映像。虚拟应用程序使应用程序发布者和IT管理员可以大大降低与软件应用程序的开发，设置，配置，部署和维护相关的成本和复杂性。


例如，基于Microsoft .NET Framework或Java运行时引擎的应用程序的发布者可以创建将应用程序与所需的运行时引擎组合在一起的虚拟应用程序。最终用户可以立即运行此应用程序，即使该用户尚未安装所需的运行时引擎或安装了不兼容的运行时引擎也是如此。这既改善了用户体验，又降低了与部署应用程序相关的测试和支持复杂性。

由于虚拟应用程序在隔离的执行环境中运行，因此可以同时执行多个应用程序，否则它们会相互干扰。例如，覆盖系统DLL或需要不同运行时引擎版本的应用程序都可以在单个主机设备上同时运行。虚拟化应用程序的另一个优势是，虚拟应用程序可以提供对特权系统资源的内部虚拟副本的访问权限，从而使没有特权的用户可以直接执行许多应用程序，而不会出现安全异常或使Vista UAC提示烦人。


与其他虚拟化系统不同，Spoon虚拟应用程序技术：


不需要任何“播放器”软件或单独安装：Spoon虚拟应用程序是可在最终用户计算机上立即运行而无需更改系统基础结构的可执行文件。

不会产生大量的处理或文件系统开销：Spoon低开销虚拟化技术允许应用程序以与本机可执行文件基本相同的性能特征运行。

不需要在虚拟应用程序上安装任何操作系统：Spoon虚拟应用程序在内部虚拟环境中提供所有必需的虚拟化操作系统功能。

Spoon虚拟化与硬件虚拟化有何不同？

与Microsoft Virtual PC和VMware等硬件虚拟化系统不同，Spoon仅虚拟化应用程序执行所需的那些操作系统功能。这使虚拟化的应用程序可以非常高效地运行，其性能特征与本机可执行文件基本相同。


Spoon虚拟化相对于硬件虚拟化的优势包括：


最佳性能：Spoon虚拟应用程序的执行速度与在主机硬件上本地运行的应用程序的速度基本相同，而仅占用很小的内存。相反，由于虚拟机必须包括整个虚拟主机操作系统，因此在硬件虚拟环境中运行的应用程序会显着降低运行速度并占用大量内存。

大大减少了应用程序的大小：Spoon虚拟应用程序所需的磁盘占用空间仅与虚拟化应用程序，数据和所包含组件的大小成比例。结果，Spoon虚拟应用程序通常足够小，可以由最终用户方便快捷地下载。硬件虚拟化需要整个主机操作系统映像，包括最终用户设备上已经存在的许多基本子系统。每个虚拟机可能占用数GB的存储空间。

能够运行多个虚拟应用程序：由于开销低，因此每个处理器可以同时运行多个Spoon虚拟环境。相反，由于硬件虚拟化的高开销，每个处理器只能运行非常少的硬件虚拟化环境。

降低许可成本：Spoon不需要购买单独的操作系统许可即可使用虚拟应用程序。硬件虚拟化系统需要主机操作系统才能正常运行，这可能会带来额外的许可成本和限制。


在某些特殊情况下，硬件虚拟化可能是合适的，包括：


非Windows操作系统：Spoon虚拟应用程序仅在Windows操作系统上运行。硬件虚拟化可以执行

与基础虚拟化硬件兼容的任何操作系统，例如Linux。

内核模式虚拟化：Spoon虚拟OS仅虚拟化用户模式操作系统功能，而硬件虚拟化系统模拟整个OS堆栈，包括内核模式组件。需要设备驱动程序或其他非用户模式软件的应用程序可能需要硬件虚拟化的环境才能正常运行。


在决定要为您的部署方案采用的技术之前，您应该仔细评估不同虚拟化方法的优缺点。

配置虚拟应用程序

虚拟化允许将文件，注册表设置，组件和其他应用程序依赖项直接嵌入到应用程序可执行文件中，从而大大简化了应用程序部署。使用Spoon虚拟化可降低设置复杂性，防止DLL冲突，并允许应用程序模拟特权磁盘和注册表资源的使用，而无需在主机上具有管理特权。

快照应用

大多数商业应用程序需要文件系统和注册表项的复杂组合才能正常运行。为了促进这些应用程序的虚拟化，Studio可以对应用程序安装进行快照，并根据在应用程序安装过程中对主机系统所做的修改自动配置它们。


快照

快照使用主机的“之前”和“之后”映像来确定虚拟应用程序配置：


“之前”快照：在安装应用程序之前拍摄。这将捕获未安装目标应用程序的主机设备的状态。

“之后”快照：安装应用程序后拍摄。这将捕获应用程序安装期间对主机设备的所有更改。然后，Studio计算快照前后的更改或增量，并将这些更改插入配置中。


要使用快照功能：


通过删除目标应用程序和所有依赖项或将Studio复制到“干净”的计算机上，准备主机设备。

虚拟化语义

如果虚拟文件系统中的文件与主机设备上实际存在的文件发生冲突，则以虚拟文件系统中的文件为准。


文件夹可以“ 完全”，“ 合并”，“ 写入副本 ”或“ 隐藏”模式虚拟化：


完全模式：即使主机设备上存在相应的目录，应用程序也只能看到虚拟文件系统中的文件，并且写入操作将重定向到沙箱数据区域。当需要完整级别的虚拟应用程序隔离时，通常使用完全模式。

合并模式：虚拟文件夹中存在的文件将与主机上相应目录中的文件合并（如果存在）。对主机文件的写入将传递到主机设备，对虚拟文件的写入将重定向到沙箱数据区域。当需要与主机设备进行某种程度的交互时，通常使用合并模式。例如，合并模式可能用于允许虚拟化的应用程序写入主机设备的“ 我的文档”文件夹。

写入复制模式：虚拟环境对主机设备上存在的文件可见，但是对文件夹内容的任何修改都将重定向到沙箱数据区域。当虚拟应用程序需要从主机设备上已经存在的文件中读取但仍然需要隔离主机设备时，通常使用写复制模式。

隐藏模式：启用“隐藏”隔离的文件和文件夹将在运行时向应用程序返回“找不到文件”消息。这适用于虚拟文件系统和主机文件系统。当主机上的文件可能会干扰应用程序正常运行的能力时，通常使用隐藏模式。通过在启用隐藏隔离的情况下将文件或文件夹添加到虚拟包，即使主机上存在文件或文件夹，应用程序也会收到“找不到文件”消息。


提示：要将选定的隔离模式应用于所有子文件夹，请右键单击该文件夹，选择“ 隔离”，单击“ 应用于子文件夹 ”复选框，然后单击“ 确定”。

可以选择从Shell浏览对话框和枚举虚拟目录内容的其他应用程序中隐藏文件和文件夹。这通常用于防止内部组件和数据文件显示给用户。要隐藏文件或文件夹，请单击所需文件或文件夹旁边的“ 隐藏”列中的复选框。

注意：不应将隐藏标志与隐藏隔离模式混淆。启用隐藏标志只会阻止在浏览对话框或目录枚举API中显示文件或文件夹。它不会阻止应用程序（因此可能是最终用户）通过直接绑定访问文件夹或文件内容。为了防止应用程序找到文件或文件夹，应启用隐藏隔离模式。

沙箱放置注意事项

在配置沙箱位置时，请注意以下建议做法：


默认情况下，沙箱位于@ APPDATALOCAL @ \ Spoon \ Sandbox \ @TITLE @ \ @ VERSION @文件夹中，其中@ APPDATALOCAL @令牌代表本地Application Data文件夹，@ TITLE @和@ VERSION @代表应用标题和版本。在属性中配置应用程序标题和版本区域。建议此位置为沙箱内容的默认位置，因为最终用户在标准Windows配置上对此位置具有完全权限。请注意，默认情况下，相同虚拟应用程序的不同版本使用相同的沙箱位置；如果不应在虚拟应用程序更新之间保留持久的用户设置，则可能需要修改此行为。


发布虚拟应用程序的新版本时，如果希望将用户设置和数据保留在新版本中，请将沙箱定向到与旧版本相同的位置。如果要重置用户设置和数据，请将沙箱定向到其他位置（通常向前滚动子目录版本号）。


如果在USB设备上部署虚拟应用程序，请将沙箱放置在@ APPDIR @目录的子文件夹中，该子文件夹表示虚拟应用程序可执行文件的位置。这将具有直接写入USB设备的效果。USB部署的建议沙箱位置为：

@ APPDIR @ \勺\沙盒\ @TITLE @ \ @ VERSION @


如果在Intranet文件共享上部署虚拟应用程序，请将沙箱放置在共享网络驱动器上的用户可访问子文件夹中。Intranet部署的建议沙箱位置为：

\\服务器名\共享\％USERNAME％\勺\沙盒\ @TITLE @ \ @ VERSION @


通常，不应将沙箱放置在任何特权文件夹下，例如@ WINDIR @或@ PROGRAMFILES @。如果Spoon引擎在运行时无法写入沙箱位置，则虚拟应用程序可能无法正确执行。


通过将变量括在百分号（即％VARIABLE％）之间，可以在沙箱位置内引用环境变量。

@ PUBLISHER @ ：产品发布者

@ VERSION @ ：完整版本字符串，点分四进制格式

Studio包含生成Microsoft Windows Installer（MSI）安装程序包的功能，以促进虚拟化应用程序的部署。除了将虚拟应用程序可执行文件部署到主机文件系统之外，Studio生成的MSI软件包还允许创建桌面和“开始”菜单快捷方式，创建与虚拟化应用程序的外壳文件扩展名关联以及用于应用程序清理的控制面板卸载程序。


本节介绍MSI安装程序包的配置和构建过程。


配置包信息

本节介绍了全局MSI软件包配置选项。这些选项位于“ 设置”设置窗格的“ MSI根树”节点上。


设定MSI套件位置

Studio生成的MSI软件包将被写入“ 输出位置”文本框中指定的文件。此文本框应包含所需输出文件的标准名称，包括输出路径和MSI文件名。


默认情况下，重建虚拟应用程序时不会自动生成或更新MSI软件包。


要在重建虚拟应用程序后自动更新MSI软件包，请选中“ 成功构建应用程序后自动生成MSI”选项。


重新生成MSI可能会大大增加完成构建过程所需的时间。因此，Spoon建议在虚拟应用程序开发过程中禁用此选项。

也可以手动强制重新生成MSI程序包。要手动生成MSI程序包，请单击“ 生成MSI”按钮。


注意：必须先生成虚拟应用程序可执行文件，然后才能生成MSI软件包。如果尚未构建虚拟应用程序可执行文件，则“ 构建MSI”按钮将被禁用。


指定包元数据

MSI安装程序包包含描述产品名称，版本（只能是数字和句点）以及制造商的程序清单。要配置MSI软件包元数据，请在“ 产品信息”区域中输入适当的值。


注意：在MSI程序包上发布的元数据与在虚拟应用程序可执行文件本身上发布的元数据不同。要修改可执行程序外壳元数据，请在“ 设置”窗格上指定适当的元数据。


安装参数

“ 安装参数”区域允许配置安装选项，例如安装位置和权限参数。

为所有用户安装应用程序选项。


注意：为所有用户安装应用程序需要对主机设备的特权访问。如果MSI软件包是为具有标准用户权限的最终用户设计的，则不要启用此选项。

如果用户在已经安装了应用程序版本的设备上运行安装程序包，则可以将MSI程序包设计为更新现有应用程序版本或与现有应用程序版本并排安装。

创建桌面和开始菜单快捷方式

桌面和开始菜单快捷方式允许最终用户直接从Windows Shell启动应用程序。要添加桌面快捷方式：

单击上桌面下节点快捷方式在节点MSI树视图。这将显示桌面快捷方式列表。

单击添加快捷方式按钮，然后选择所需的快捷方式名称，目标和选项。“ 目标”下拉列表中填充了启动文件列表，使快捷方式可以快速连接到自动存储塔入口点。

SpoonReg注册工具简介

SpoonReg是一种工具，提供了用于部署虚拟应用程序和管理虚拟桌面环境的简单命令行界面。用户和管理员可以使用SpoonReg为单个用户或一组用户或设备注册虚拟应用程序。SpoonReg可用于部署和管理使用Spoon Studio构建的虚拟应用程序和层。


用Spoon Studio虚拟化应用程序后，通常需要使应用程序的“开始菜单”图标，快捷方式和文件关联在用户的桌面上可用。SpoonReg允许您在外壳中注册Spoon虚拟应用程序，从而创建通常在标准安装过程中创建的所有外壳关联。但是，与执行安装不同，注册和注销几乎可以立即执行。


SpoonReg还提供创建，重置和删除应用程序沙箱的功能-虚拟化应用程序所在的虚拟环境“冒泡”。沙盒管理提供对应用程序链接和互通的细粒度控制。


Spoon Server用户和管理员可以使用与Spoon Server关联的SpoonReg机制将应用程序注册到桌面。对于特殊的部署方案，请联系您的Spoon代表以了解如何获取自己的SpoonReg.exe实用程序版本。


使用SpoonReg注册虚拟应用程序

SpoonReg提供了用于管理虚拟桌面环境的简单命令行界面。本节介绍SpoonReg命令行基本语法，包括注册，更新和注销虚拟应用程序的步骤。


命令行语法

本节中使用以下命名约定：


参数

描述

AppSpec

AppSpec是使用Spoon Studio构建的虚拟可执行文件或层的路径（相对或完全限定）

SandboxSpec

SandboxSpec是虚拟沙箱的名称或路径


注册虚拟应用程序

要注册应用程序，请使用以下命令：

SpoonReg.exe应用程序规格


此命令创建与虚拟应用程序可执行文件关联的所有“开始”菜单项，桌面快捷方式以及文件关联。


默认情况下，注册将创建虚拟应用程序可执行文件的本地缓存副本，并将用户的本地配置文件用作沙箱位置。


注意：使用SpoonReg工具或Spoon虚拟桌面服务注册应用程序时，将忽略在虚拟应用程序构建期间指定的沙箱位置。

命令行参数可用于控制应在其中注册虚拟应用程序的缓存行为和沙箱：

SpoonReg.exe [选项] AppSpec [@SandboxSpec]

行为

/非缓存

SandboxSpec

此参数引用现有沙箱的名称和路径。如果指定了此参数并且存在具有该名称的沙箱，则该应用程序将被注册到该沙箱中。（有关其他详细信息，请参阅本节中的“沙箱管理”主题。）

SpoonReg.exe [Option] AppSpec [@SandboxSpec]

行为

/高速缓存

启用指定应用程序的缓存（反转/ nocache设置）


注销虚拟应用程序

取消注册虚拟应用程序会逆转注册过程，删除虚拟应用程序，“开始”菜单图标，快捷方式和文件关联。


要注销虚拟应用程序，请使用以下命令：

SpoonReg.exe /取消注册AppSpec [@SandboxSpec]


也可以使用单个命令注销所有应用程序：

SpoonReg.exe /取消注册


客户端配置文件

SpoonReg命令可以应用于“ 本地”，“ 所有用户”或“ 漫游 Windows”配置文件。这些配置文件对应于Windows操作系统上可用的用户配置文件。建议将SpoonReg命令尽可能应用于所有用户配置文件。


本地资料

Windows设备上的每个用户都有一个本地用户配置文件。本地配置文件的任何更改仅影响该设备上的该用户。如果在SpoonReg命令行上未明确指定任何配置文件，则本地配置文件是SpoonReg使用的默认配置文件。


所有用户个人资料

每个设备都有一个“ 所有用户”配置文件。对“ 所有用户”配置文件所做的任何更改都会影响设备上的所有用户。


要将应用程序注册到“ 所有用户”配置文件，请使用/ allusers命令行标志执行SpoonReg命令。


您必须在设备上具有管理权限才能将应用程序注册到“ 所有用户”配置文件。


漫游个人资料

Active Directory环境中的每个用户都可以具有一个漫游配置文件，该配置文件根据目录策略镜像到其他计算机。通常，漫游配置文件存储在网络服务器上，并且可从网络上的所有设备获得。


要将应用程序注册到漫游配置文件，请使用/ roaming标志执行SpoonReg命令。


注意：没有所有用户的漫游配置文件。因此，/ roaming标志与/ allusers标志一起使用时没有影响。


注意：由于SpoonReg应用于特定的用户配置文件，因此不能用于LocalSystem帐户（没有关联的配置文件）。


沙箱管理

SpoonReg工具允许创建和管理一个或多个虚拟环境沙箱。


甲沙箱包含所有的虚拟应用程序的分离的数据和设置的作为通过所述虚拟应用程序的隔离配置设置来确定。注册到同一沙箱的应用程序可以查看和修改彼此的虚拟化数据和设置。


默认情况下，所有应用程序都注册到名为Default的单个默认沙箱中。在某些情况下，可能需要将相关应用程序分组到可以视为单个管理单元的沙箱中。重置沙箱后，将清除该沙箱中存储的所有应用程序内容和数据，并恢复为默认状态。


创建一个沙箱

如果在注册过程中未指定沙箱，则该应用程序将注册到默认沙箱（Default）。要创建其他沙箱，请使用以下命令之一：

SpoonReg.exe [配置文件] / create [SandboxName] [SandboxPath] SpoonReg.exe [Profile] / c [SandboxName] [SandboxPath]


如果未提供路径，则会在指定配置文件下的AppData文件夹下创建默认路径。


重置沙箱

重置沙箱会撤消对沙箱所做的所有更改，包括用户对数据或设置所做的任何更改。这会将所有注册到沙箱的应用程序恢复为其默认状态。


要重置沙箱，请使用以下命令之一：

SpoonReg.exe [配置文件] / reset [SandboxSpec] SpoonReg.exe [配置文件] / r [SandboxSpec]


如果未提供SandboxSpec，则将重置默认沙箱。


删除沙箱

删除沙箱会从沙箱中删除所有应用程序，数据和设置。要删除沙箱，请使用以下命令之一：

SpoonReg.exe [配置文件] /删除[SandboxSpec] SpoonReg.exe [配置文件] / d [SandboxSpec]


如果未提供SandboxSpec，则将重置默认沙箱（无法删除默认沙箱）。注册到已删除沙箱的所有应用程序都将移至默认沙箱。


移动沙箱

您可以使用SpoonReg将沙箱位置移动到给定路径。移动沙箱：

SpoonReg.exe [配置文件] / move [SandboxSpec] [SandboxPath]

活动目录

Active Directory允许网络管理员管理组织内的用户和组。许多组织使用Active Directory来管理其网络服务。通过将Active Directory与SpoonReg结合使用，管理员可以轻松，可靠地向其组织中的一个或多个用户部署虚拟应用程序。

将SpoonReg与Active Directory一起使用

在组织中，通常更希望管理一组用户，而不是一次管理一个用户。通过将Active Directory与SpoonReg结合使用，您可以为用户，组或组织单位管理虚拟环境。在这里，我们将介绍一些有关如何实现此目标的方案。

方案1：将组织单位（OU）链接到组策略对象（GPO）

Active Directory提供了许多方法来管理组织的网络服务。在这里，我们将研究如何结合使用OU和GPO来为一组用户管理虚拟应用程序环境。


组织单位（OU）

在Active Directory中，OU是可以放置用户，组和其他OU的容器。管理员可以使用这些容器创建对组织内的层次结构或逻辑结构建模的结构。通过设置OU，我们可以隔离将接收自己的虚拟应用程序集的不同用户组。例如会计，销售，市场营销等。


组策略对象（GPO）

GPO是将一组规则和功能应用于目标用户的一种方式。通常，GPO处理安全性，应用程序安装，登录/注销脚本，Internet Explorer设置等。当用户登录到给定域时，通常会应用GPO。根据它们的配置文件，将应用各种GPO。


SpoonReg和GPO

为了部署虚拟应用程序，我们可以将GPO配置为运行必要的SpoonReg命令以注册给定的一组虚拟应用程序。您可以使用组策略对象编辑器创建和编辑GPO 。这是Microsoft Active Directory的一个附件。处理虚拟应用程序管理的最简单方法是创建登录脚本，该脚本为组织中的特定组注册虚拟应用程序。您可以在组策略对象编辑器的“用户配置”>“ Windows设置”>“脚本”下执行此操作。您可以为会计组添加以下登录脚本：

\\ VirtualAppServer \ Tools \ SpoonReg.exe \\ VirtualAppServer \ AllVirtualApps \ Firefox.exe

\\ VirtualAppServer \ Tools \ SpoonReg.exe \\ VirtualAppServer \ AllVirtualApps \ AcrobatReader.exe


而您可以为图形设计组添加以下内容：

Spoon虚拟桌面服务简介

Spoon虚拟桌面服务在用户的计算机上运行，​​并将其虚拟桌面环境与中央SMB文件服务器同步。使用/ allusers标志注册虚拟应用程序或客户端配置文件时，将自动安装该服务。


创建客户端配置文件

客户端配置文件（.xclient文件）提供了一种在单个文件中管理用户所有虚拟应用程序的方法。使用SpoonReg注册虚拟应用程序时，该注册信息存储在默认客户端配置文件中。


建议使用虚拟桌面服务的方法是使用您要分发的所有虚拟应用程序设置一台计算机（请参阅“在Windows Shell中注册虚拟应用程序”）。来自本机的客户端配置文件将用作主配置文件。您想要具有相同虚拟桌面配置的所有用户都使用虚拟桌面服务与该文件同步。


要将应用程序注册到初始客户端，请使用以下命令：

SpoonReg.exe应用程序规格


注意：请记住，虚拟应用程序在网络上的位置对于正在访问这些应用程序的所有用户必须是可访问的并且一致。


发布客户端配置文件

创建主客户端配置文件后，使用以下命令之一将其发布到所需客户端计算机可以访问的网络共享中：

SpoonReg.exe / publish PublishPathSpoonReg.exe / p PublishPath


注意：Spoon虚拟桌面服务作为LocalSystem运行，这意味着必须仔细考虑安全性。LocalSystem可以通过计算机帐户访问其域中的资源。机器帐户名称是机器名称，后跟$（例如，服务器的机器帐户是server $）。计算机帐户也包括在“所有人”域组中。最终的后果是，远程应用程序源所在的网络位置必须是对所有人或特定计算机帐户具有读取访问权限的网络共享。它不能是映射的网络驱动器，因为映射的驱动器处于单个用户级别。


注册客户端配置文件

发布主配置文件后，使用以下命令在将共享相同虚拟桌面环境的计算机上注册它：

SpoonReg.exe / allusers ClientConfigurationPath

注意：您必须是管理员才能使用/ allusers标志。


注册客户端配置文件将自动安装并启动Spoon虚拟桌面服务并同步虚拟桌面环境。


更新默认轮询间隔

Spoon虚拟桌面服务的默认轮询间隔为60秒。您可以使用SpoonReg通过以下命令在单个计算机上更新此间隔：

SpoonReg.exe / updateinterval [秒]


使用零或零作为参数几秒钟将禁用自动更新。


在应用程序级别禁用自动更新

要禁用单个应用程序的自动更新，您需要更新主客户端配置文件。以下命令更新文件，然后应将其重新发布到共享位置：

SpoonReg.exe / noupdate / allusers ApplicationPath

疑难解答

本节介绍使用Studio时发生的最常见的配置错误。


如果您在使用虚拟应用程序时遇到问题，请在使用其他支持选项之前仔细阅读本节或查询在线知识库。这些地方之一很可能会解决您遇到的问题。


问题访问基于互联网的资源

Studio的某些功能需要访问基于Internet的资源才能正常运行。如果Studio无法连接到Internet，则这些功能可能不可用。


在许多公司环境中，通过防火墙或代理服务器过滤对Internet的访问。在这种情况下，Studio会尝试根据系统Internet设置自动将其自身配置为可以访问Internet。但是，在某些情况下，可能需要手动配置代理服务器设置。


要配置代理服务器设置：


单击选项菜单上的代理设置...选项。这将显示“ 代理设置”对话框。

在对话框中输入适当的代理服务器设置。您可能需要咨询系统管理员，以获取组织的代理服务器设置。


生成诊断模式虚拟应用程序

有时，虚拟应用程序配置期间的错误会导致可执行文件无法正常运行。也就是说，要精确地模拟原始源应用程序的行为。通常，这是由于虚拟化配置错误（例如缺少文件或注册表项）导致的。


为了帮助诊断这些问题，Studio提供了创建诊断模式可执行文件的选项。诊断模式的可执行文件在执行期间生成日志记录数据，可以帮助诊断与虚拟化相关的问题。


要生成诊断模式可执行文件，请选中“ 虚拟应用程序”功能区栏的“ 输出”部分上的“ 生成诊断模式可执行文件”选项。然后单击“ 生成”以生成检测的可执行文件。


检测到的可执行文件的执行将在应用程序启动目录中生成xclog_ <id> .txt文件，该文件包含在执行期间收集的详细诊断数据。检查此文件，尤其是检查标记为WARNING或ERROR的条目，通常可以诊断虚拟化错误。如果您需要Spoon技术支持的帮助来解决问题，我们强烈建议您将这些信息与支持请求一起提交，以帮助解决问题。


注意：由于诊断模式的可执行文件比标准可执行文件的运行速度明显慢，并且会生成非常大的日志文件，因此不应将诊断模式的可执行文件分发给最终用户。
